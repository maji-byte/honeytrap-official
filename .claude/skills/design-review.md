# Design Review Skill

## Description
HoneyTrapサイトのデザイン一貫性をチェックするレビュースキル。
ブランドガイドライン・デザインシステムへの適合性を自動検証する。

## Trigger
`/design-review` または デザイン関連の変更をレビューする際

## Steps
1. 変更対象ファイルを特定（git diff または指定ファイル）
2. 以下のドキュメントを参照:
   - `/docs/01_brand_concept.md` — ブランドDO/DON'T
   - `/docs/04_visual_design_system.md` — デザイントークン
3. チェック項目:
   - [ ] カラーがデザイントークンに存在するか（ハードコードされた色がないか）
   - [ ] タイポグラフィが規定フォント・サイズを使用しているか
   - [ ] スペーシングが8px基準のトークンを使用しているか
   - [ ] 画像比率が規定に沿っているか
   - [ ] アニメーションが許可リストに含まれるか
   - [ ] 純黒(#000000)が使用されていないか
   - [ ] box-shadowが使用されていないか
4. 結果をレポート形式で出力

## Output
```
## デザインレビュー結果
- 対象: [ファイルリスト]
- 適合: ✅ / ⚠️ 要修正
- 詳細: [チェック項目ごとの結果]
- 修正提案: [具体的なコード修正]
```
